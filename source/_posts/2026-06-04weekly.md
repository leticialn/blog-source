## 主要收获

- 对“大模型提升强化学习”概念的理解：把大语言模型或多模态基础模型当作“增强器”来提升强化学习系统的表示、奖励、策略、规划、样本效率和泛化能力，或者反过来用强化学习去提升大模型的对齐、推理和智能体能力。从研究视角看，它不是单一算法，而是一类“LLM + RL”的交叉范式：大模型可以充当策略网络、奖励模型、世界模型、验证器、规划器、记忆模块或环境接口，从而改变 RL 的学习方式。

- 主要研究路径包括：
  - 用 RL 提升大模型：典型是 RLHF、DPO、GRPO、RLVR 等方法，用奖励或偏好信号去提升大模型的回答质量、推理能力和行为可控性。

  - 用大模型增强 RL（LLM for RL）：让大模型参与 RL 的关键模块，如表征器、奖励建模器、世界模型/规划器、验证器/审稿人。

  - 让 RL 进入 agent 场景：把 LLM 视为在动态环境中行动的 agent，用 RL 训练它做搜索、写代码、操控 GUI、完成数学证明或执行多步任务。

- 当前热点：

  - 推理增强：重点是让模型通过 RL 学会更长链条的中间推理，常见任务是数学和代码，目标是提高复杂问题求解能力。

  - 过程级奖励：从只看最终答案，转向对“思考过程”打分，例如过程奖励模型（PRM）和 verifier-guided training，这类路线被认为更适合复杂 reasoning。

  - 工具与搜索：让 LLM 学会何时搜索、如何修正查询、如何整合证据，是 agentic RL 里增长最快的方向之一。

  - 长程环境交互：GUI agent、代码 agent、网页 agent、机器人/具身智能都在把 RL 引入更真实的交互环境中。

- 代表方法：

|     方向     |        代表方法/范式        |                        核心特点                        |
| :----------: | :-------------------------: | :----------------------------------------------------: |
|   偏好对齐   |            RLHF             |       用人类反馈训练奖励模型，再用 RL 优化策略。       |
| 简化偏好优化 |       DPO / SimPO 类        |          通过直接偏好优化减少传统 RL 复杂度。          |
| 组内相对优化 |            GRPO             |       常用于推理任务，强调组采样与相对优势估计。       |
| 结果导向推理 |  RLVR / verifier-based RL   | 用可验证结果作为奖励，尤其适合数学、代码和可判定任务。 |
|  Agentic RL  | 长程 POMDP + 工具/记忆/规划 |          从文本生成升级为环境中的决策与行动。          |

- 主要瓶颈：
  - 算力和样本成本高：RL 尤其在 LLM 上训练代价大，采样、rollout、critic 计算都昂贵。
  - 奖励黑客和过优化：模型可能学会投机而不是掌握真正能力，这是 RLHF 和 reasoning-RL 的共性问题。
  - 评测难：很多 agent 任务缺少统一、稳定、可复现的 benchmark，导致不同方法很难公平比较。
  - 环境构建不足：长程交互需要高质量环境、工具链和轨迹数据，但这类资源仍偏稀缺。
- 可提升的 RL 要素：
  - 奖励：构造更细粒度、更语义化的奖励，解决奖励稀疏、难设计、易被 hack 的问题。
  - 策略：大模型可直接作为策略或帮助策略更好地表达复杂动作分布。
  - 价值函数：帮助价值评估在文本、代码等高维空间里变得更语义化。
  - 模型：充当环境/世界模型，提前估计动作后果，提升规划和样本效率。
  - 探索：大模型可以显著改善探索质量，引入“带先验的结构化探索”。
  - 信用分配：利用中间步骤的可解释性、可验证性，改善长链任务的信用分配。

## 遇到的问题

- 知识面宽泛，学到的内容过于碎片化，粗浅且泛泛。
- 好像调研方向有点偏差，阅读的资料主要是“用 RL 提升大模型”而不是“用大模型增强 RL”的算法类问题。
