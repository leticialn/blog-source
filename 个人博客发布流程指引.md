## ✍️ 一、新建文章

在终端中，进入博客根目录后，使用对应的 Hexo 脚手架命令新建文章。Hexo 会自动套用模板（位于 `scaffolds/` 目录）并正确设置分类与标签。

```bash
cd "/Users/zoey/Desktop/File/个人博客/my-blog"
```

```bash
# 📅 周报总结
npx hexo new weekly "第 X 周研究周报：[主题]"

# 📄 论文精读
npx hexo new paper "论文名称精读"

# 🧪 实验日志
npx hexo new experiment "实验名称"

# 📝 学习笔记
npx hexo new note "笔记主题"
```

新建的文章文件位于 `source/_posts/` 目录下，用编辑器（VS Code 等）打开后即可按模板填写内容。

## ⚠️ 二、写作注意事项（避免之前踩过的坑）

### 1. 中文字体

已在主题 CSS 中统一配置好仿宋字体栈，无需手动设置。博客中所有中文内容都会自动渲染为仿宋体（macOS 显示为华文仿宋 STFangsong）。

### 2. 图片插入

❌ **不要** 直接引用 Gitee 外链图片，Gitee 有防盗链限制，图片会无法显示：
```markdown
<!-- 错误：Gitee 外链 -->
![图片](https://gitee.com/xxx/xxx/raw/master/img/xxx.png)
```

✅ **正确方式**：将图片文件下载保存到 `source/images/` 目录下，再用相对路径引用：
```bash
# 在终端中使用 curl 下载图片
curl -L -o source/images/your-image-name.png "图片URL"
```
```markdown
<!-- 正确：本地相对路径 -->
![图片描述](/images/your-image-name.png)
```

### 3. 表格格式

已在主题 CSS 中添加了完整的表格样式（边框、斑马纹），表格按照标准 Markdown 语法写即可正常显示：

```markdown
| 列一 | 列二 | 列三 |
|:---:|:---:|:---:|
| 内容 | 内容 | 内容 |
```

## 🚀 三、手动推送发布（写完文章后执行）

### 步骤 1：编译并部署到 GitHub Pages

```bash
npx hexo clean && npx hexo generate && npx hexo deploy
```

执行成功后，文章自动发布到 [https://leticialn.github.io](https://leticialn.github.io)，同时 `LIST.md` 会被自动更新。

### 步骤 2：将源码与 LIST.md 同步备份到 blog-source 仓库

```bash
# 将 LIST.md 同步为源码仓库的 README
cp LIST.md README.md

# 提交并推送全部源码变更
git add .
git commit -m "✍️ 新增文章：[文章标题]"
git push origin main
```

### 完整一键命令（推荐日常使用）

```bash
npx hexo clean && npx hexo generate && npx hexo deploy && cp LIST.md README.md && git add . && git commit -m "✍️ 更新博客内容 $(date '+%Y-%m-%d')" && git push origin main
```

## 🤖 四、每日零点自动更新（GitHub Actions）

项目已配置 GitHub Actions 工作流（`.github/workflows/auto-deploy.yml`），系统会**每天北京时间 00:00 自动触发**，完成以下操作：

1. 拉取最新源码；
2. 执行 `hexo clean && hexo generate` 编译；
3. 执行 `hexo deploy` 推送到 GitHub Pages；
4. 将最新 `LIST.md` 内容同步到 `README.md` 并推送到源码仓库。

### 首次启用前需要做的配置 ⚠️

GitHub Actions 在服务器上运行，需要一个 SSH 密钥来代替你的身份推送到 `leticialn.github.io` 仓库，步骤如下：

**第一步：生成专用部署密钥（在你的 Mac 本地终端执行）**

```bash
ssh-keygen -t ed25519 -C "hexo-deploy-key" -f ~/.ssh/hexo_deploy_key -N ""
```

**第二步：将公钥添加为 `leticialn.github.io` 仓库的 Deploy Key**

```bash
# 复制公钥内容
cat ~/.ssh/hexo_deploy_key.pub | pbcopy
```

1. 打开 [https://github.com/leticialn/leticialn.github.io/settings/keys](https://github.com/leticialn/leticialn.github.io/settings/keys)
2. 点击 "Add deploy key"，粘贴公钥内容，勾选 "Allow write access"。

**第三步：将私钥添加为 `blog-source` 仓库的 Secret**

```bash
# 复制私钥内容
cat ~/.ssh/hexo_deploy_key | pbcopy
```

1. 打开 [https://github.com/leticialn/blog-source/settings/secrets/actions](https://github.com/leticialn/blog-source/settings/secrets/actions)
2. 点击 "New repository secret"，Name 填 `DEPLOY_KEY`，将私钥粘贴进去保存。

配置完成后，每天零点自动部署就会生效，你也可以在仓库的 **Actions** 标签页点击 "Run workflow" 手动触发。

## 📋 五、LIST.md 目录说明

`LIST.md` 是博客的自动生成目录文件，由 `scripts/generate_list.js` 脚本在每次执行 `hexo generate` 时自动更新，**无需手动维护**。

文件更新后，通过 `cp LIST.md README.md` 命令将其同步为源码仓库的 README 首页，访问 [https://github.com/leticialn/blog-source](https://github.com/leticialn/blog-source) 即可看到最新目录。
